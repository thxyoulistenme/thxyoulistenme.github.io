程序员的三门课：技术精进、架构修炼、管理探秘

阅读途径: 微信读书
显示页数: 585页
阅读进度: 210/585

2020年2月15日 阅读进度 210/585

收获1: 写代码前先写好概要设计以及单元测试

在需求评审后的那一段时间,是对需求了解最深的时候.不必急于求成直接上手去写代码.根据自己对需求的理解,先按照思路输出概要设计和编写单元测试.这是一个需求固化的过程,也是个能发现需求风险点的阶段.直接上手代码可能写着写着忘记了某些微小点.

收获2: 测试困难时,要有去写工具简化的想法

有时候因为对接或者环境问题,导致测试无法进展或者重复性工作.再遇到这些问题时,要有去解决的想法.通过编写工具,提升测试效率.同样也能避免多次重复工作导致厌烦.而且编写工具的过程也是一个成长的过程.

收获3: 一些比较好用的工具

核心主要以下几个网站

  • http://tool.chinaz.com
  • https://tool.oschina.net
  • https://www.107000.com/

1. 静态代码分析工具

可以用来扫描代码,提前发现一些代码漏洞和语法不规范的地方

  • findbugs
  • P3C

2. 生成类在线工具

  • sql自动生成java代码 https://www.devtool.com/javacodegen.html
  • json生成java类 https://www.bejson.com/json2javapojo/new/
  • corn表达式生成 http://cron.qqe2.com/
  • 正则代码生成 http://tool.chinaz.com/tools/regexgenerate

3. 转换类在线工具

  • 编码转换
    • http://tool.chinaz.com/Tools/unicode.aspx
    • https://www.107000.com/T-Hex
  • 时间戳转换 http://tool.chinaz.com/Tools/unixtime.aspx
  • 加密/解密
    • http://tool.chinaz.com/Tools/md5.aspx
    • https://www.somd5.com/
  • 单位换算 http://tool.chinaz.com/Tools/angle

4. 检查类在线工具

  • json在线验证及格式化 http://www.bejson.com/index.html
  • 正则验证 http://tool.chinaz.com/regex

5. 对照类在线工具

  • ASCII对照 https://tool.oschina.net/commons?type=4
  • HTTP状态 https://tool.oschina.net/commons?type=5
  • HTTP Content-type https://tool.oschina.net/commons
  • TCP/UDP常见端口 https://tool.oschina.net/commons?type=7
  • HTML转义字 https://tool.oschina.net/commons?type=2
  • RGB颜色 https://tool.oschina.net/commons?type=3
  • 网页字体 https://tool.oschina.net/commons?type=8

6. 其他在线工具

  • 在线UML制图 https://www.processon.com

收获4 做好代码审查,善于分享,多重构

自己写好的代码,从自己的角度来看,可能看不到问题.通过代码审查的这一方式,可以从别人的角度来发现问题.并且感觉写的比较好的代码、使用到了比较方便的api、解决了棘手的问题等都可以向他人分享,自己学会不算会,给他人讲解并能让他人懂才是真正的掌握.代码写完几个月后回头看,可能会看到漏洞百出,如果可以的话,可以着手重构.
